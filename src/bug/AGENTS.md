# bug LLM 索引

## 适用范围

`src/bug/` 负责 Bug 分类、报告和跟踪。

## 设计规范

- Bug 数据应包含复现步骤、实际结果、期望结果、严重程度、影响范围和关联执行 ID。
- 分类逻辑应可解释，便于 fixer、reviewer 和 tester 接续处理。
- 报告输出应结构化，适合写入 issue、日志或 UI。

## 约束

- 不要把敏感日志、token、用户数据原文直接写入 bug report。
- 不要把未验证猜测标记为根因；区分现象、假设和证据。
- 外部 issue tracker 写入必须由 integration 层处理。

## 最佳实践

- 保留原始错误摘要和最小复现上下文。
- 分类优先使用稳定枚举，避免 UI 和查询依赖自由文本。
- 为分类边界、空报告和多错误聚合写测试。
