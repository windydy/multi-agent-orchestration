# integrations LLM 索引

## 适用范围

`src/integrations/` 放置外部系统集成适配，包括 GitHub、Jira、Slack 等。

## 设计定位

- 每个文件封装一个外部系统的 API 细节和认证方式。
- 上层 workflow/executor 只调用领域方法，不直接拼接外部 API 请求。
- 集成层负责错误归一化、限流处理和响应解析。

## 约束

- 不要硬编码 token、workspace id、项目 key、频道名或组织私有 URL。
- 所有外部输入和响应都视为不可信，解析前做字段校验。
- 网络错误、认证失败、权限不足、速率限制要区分处理。
- 会产生外部副作用的操作必须让调用方显式选择，不能在读取方法中隐式写入。

## 最佳实践

- 客户端构造接受 token/base_url 参数或从环境读取，并在缺失时快速失败。
- 返回项目内部 DTO 或 dict，避免把第三方 SDK 对象传出集成层。
- 对外部 API 调用加超时和重试策略，但不要无限重试。
- 测试中使用 fake client 或 fixture 响应，避免真实调用第三方服务。
