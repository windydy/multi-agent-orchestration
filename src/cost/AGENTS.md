# cost LLM 索引

## 适用范围

`src/cost/` 负责模型调用和工作流执行的成本控制。

## 设计规范

- 成本控制应以 warning、limit、stop 等阈值表达渐进策略。
- 成本数据要带时间、thread_id、node/agent 标识和模型信息。
- 预算判断应在模型调用前后都可执行：调用前预估，调用后记账。

## 约束

- 不要把成本控制写成只记录不阻断；达到 stop 阈值必须让上层能停止执行。
- 不要假设单一模型价格；模型价格表应可配置或隔离。
- 不要在错误消息中泄露 prompt 或密钥。

## 最佳实践

- 对接 observability，将 24h 成本、token 和节点成本聚合展示。
- 成本阈值默认保守，并允许 workflow 配置覆盖。
- 测试覆盖 warning、limit、stop 和未知模型路径。
