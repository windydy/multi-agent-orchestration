# tools LLM 索引

## 适用范围

`src/tools/` 放置供 Agent/Executor 调用的领域工具，如架构、CI/CD、数据分析、依赖审计、Docker、PM、安全扫描和 SQL 工具。

## 设计规范

- 工具应是小而明确的能力单元，输入输出结构化。
- 工具只做被请求的动作，不自行决定 workflow 路由或任务范围扩张。
- 高风险工具默认保守，优先 dry-run、计划、检查，再执行有副作用操作。

## 约束

- shell、文件、网络、数据库和部署操作必须验证输入并限制作用域。
- 不要在工具中硬编码凭据、默认生产地址或危险命令。
- SQL 工具必须使用参数化查询或只生成待审查 SQL，不拼接不可信输入。
- 安全扫描工具不得提供规避检测或破坏性攻击流程。

## 最佳实践

- 新工具先定义清晰的参数模型和返回模型。
- 失败返回应包含可操作错误信息和是否可重试。
- 工具结果要适合写入日志和 UI，避免返回巨大原始输出。
- 为危险输入、空输入和外部失败路径写测试。
